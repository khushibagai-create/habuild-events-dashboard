# Habuild Mobile App: Full Analytics Event Payloads

Every `trackEvent(...)` call site in the app, with the complete payload (actual button name, screen value, and the value of every property). Generated by a balanced-paren source parser over `app/**/*.ts` and `app/**/*.tsx` (tests, mocks, styles, and stories excluded). Companion CSV: `app-events-payloads.csv`.

## Summary

- Total call sites: 336
- Payload values captured: 940 across all call sites
- Literal values (resolved statically): 616 (65.5 percent)
- Dynamic values (a variable, member expression, or template, so the real value is only known at runtime): 324 (34.5 percent)
- Dynamic values are marked with a leading `~` in the payload column. Ternary-of-literals values are fully resolved and shown as `a | b`.
- The 36 `log` wrapper calls are resolved to their real event via the `event_name` key, so the event column always shows the true event.
